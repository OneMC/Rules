# PRD Skill - 产品需求文档

为 iOS 项目提供标准化的产品需求文档编写规范和工作流。

## 使用场景

- 创建新功能的 PRD
- 评审和优化现有 PRD
- 将需求拆解为技术任务
- 编写用户故事和验收标准

## PRD 文档结构

标准 PRD 应包含以下章节：

### 1. 背景与目标
- 需求来源
- 业务目标
- 成功指标

### 2. 用户故事
格式：作为 [角色]，我希望 [功能]，以便 [价值]

### 3. 功能规格
- 功能清单（优先级 P0/P1/P2）
- 详细功能描述
- 输入/输出定义
- 边界情况处理

### 4. 交互设计
- 页面流程图
- 关键界面描述
- 状态与反馈

### 5. 技术约束
- 平台版本要求
- 依赖项
- 性能指标

### 6. 验收标准
可测试的通过条件

## 模板

使用 `assets/prd-template.md` 作为新文档起点。
